## 2）四类 adapter 的职责

### A. OpenCodeAdapter

只负责原生对话：

* 创建 OpenCode session
* 发消息
* 取消息历史
* 后续可扩展 SSE / async

OpenCode 官方 server 已经直接暴露了 session、message、`prompt_async`、`/event` 等接口，所以这层很适合被你自己的 BFF 二次封装。([OpenCode][1])

### B. SkillChatAdapter

本质还是走 OpenCode，只是比直聊多一层：

* session 上挂 `skill_name`
* 发消息时自动注入 `system prompt`
* 要求模型优先使用原生 `skill` 工具加载该 skill

OpenCode 官方 skill 是从 `SKILL.md` 发现并按需加载的，所以 Skill V1 不必自造一套执行器。([OpenCode][2])

### C. WebSDKAdapter

给知识库和旧应用共用：

* 生成 `launch_token`
* 记录启动记录
* 返回一个可嵌入的 html host 页或 embed config
* 不管理它自己的聊天历史

### D. OpenWorkAdapter

只负责文件型技能/config能力：

* 列出 skills
* 未来可发布 skill / reload engine
* 这部分不直接暴露给普通用户编辑

OpenWork 官方架构文档明确建议把 `.opencode/`、`SKILL.md`、`opencode.json` 这类读写统一走 server 路径，而不是让前端直接碰本地文件系统。([GitHub][3])

## 3）资源目录 + 权限模块

```python
# app/catalog/service.py
import json
from pathlib import Path
from app.models import Resource

class ResourceCatalog:
    def __init__(self, path: str = "config/resources.json"):
        self.path = Path(path)
        self._items = [Resource(**x) for x in json.loads(self.path.read_text("utf-8"))]

    def list_all(self) -> list[Resource]:
        return [x for x in self._items if x.enabled]

    def get(self, resource_id: str) -> Resource | None:
        for item in self._items:
            if item.id == resource_id and item.enabled:
                return item
        return None

```python
# app/store/redis_store.py
import time, json
from redis.asyncio import Redis
from app.models import PortalSession, LaunchRecord

class PortalStore:
    def __init__(self, redis: Redis):
        self.redis = redis

    async def save_session(self, session: PortalSession):
        key = f"portal:session:{session.id}"
        await self.redis.hset(key, mapping=session.model_dump(mode="json"))
        await self.redis.zadd(f"portal:user:{session.emp_no}:sessions", {session.id: time.time()})

    async def get_session(self, session_id: str) -> PortalSession | None:
        data = await self.redis.hgetall(f"portal:session:{session_id}")
        return PortalSession(**data) if data else None

    async def list_user_sessions(self, emp_no: str) -> list[str]:
        return await self.redis.zrevrange(f"portal:user:{emp_no}:sessions", 0, 49)

    async def save_launch(self, launch: LaunchRecord):
        key = f"portal:launch:{launch.id}"
        await self.redis.hset(key, mapping=launch.model_dump(mode="json"))
        await self.redis.zadd(f"portal:user:{launch.emp_no}:launches", {launch.id: time.time()})

    async def get_launch(self, launch_id: str) -> LaunchRecord | None:
        data = await self.redis.hgetall(f"portal:launch:{launch_id}")
        return LaunchRecord(**data) if data else None

## 五、执行层代码骨架

## 1）OpenCodeAdapter

OpenCode server 支持 `opencode serve`、CORS 白名单和 basic auth；接口上有 session、message、`prompt_async` 与 SSE 事件流，所以后端封装这层最自然。([OpenCode][1])

```python
# app/adapters/opencode.py
import httpx

class OpenCodeAdapter:
    def __init__(self, base_url: str, username: str | None = None, password: str | None = None):
        self.base_url = base_url.rstrip("/")
        self.auth = (username, password) if password else None

    async def _get(self, path: str):
        async with httpx.AsyncClient(timeout=60, auth=self.auth) as client:
            resp = await client.get(f"{self.base_url}{path}")
            resp.raise_for_status()
            return resp.json()

    async def _post(self, path: str, payload: dict, expect_json: bool = True):
        async with httpx.AsyncClient(timeout=120, auth=self.auth) as client:
            resp = await client.post(f"{self.base_url}{path}", json=payload)
            resp.raise_for_status()
            if expect_json:
                return resp.json()
            return None

    async def create_session(self, title: str | None = None) -> dict:
        payload = {}
        if title:
            payload["title"] = title
        return await self._post("/session", payload)

    async def list_messages(self, session_id: str, limit: int = 100) -> list[dict]:
        return await self._get(f"/session/{session_id}/message?limit={limit}")

    async def send_message(
        self,
        session_id: str,
        text: str,
        *,
        system: str | None = None,
        model: dict | None = None
    ) -> dict:
        payload = {
            "parts": [{"type": "text", "text": text}]
        }
        if system:
            payload["system"] = system
        if model:
            payload["model"] = model
        return await self._post(f"/session/{session_id}/message", payload)

    async def send_message_async(self, session_id: str, payload: dict) -> None:
        await self._post(f"/session/{session_id}/prompt_async", payload, expect_json=False)
```

## 2）SkillChatAdapter

```python
# app/adapters/skill_chat.py
from app.adapters.opencode import OpenCodeAdapter

def build_skill_system_prompt(skill_name: str, desc: str = "") -> str:
    return f"""
You are in skill mode "{skill_name}".
Treat this skill as the primary workflow for this session.
Use the native skill tool to load "{skill_name}" when needed.
Do not silently switch to another skill.
If the user's request is outside this skill, explain briefly and ask whether to exit skill mode.
Skill summary: {desc}
""".strip()

class SkillChatAdapter:
    def __init__(self, opencode: OpenCodeAdapter):
        self.opencode = opencode

    async def create_session(self, *, resource: dict, user: dict, trace_id: str) -> dict:
        return await self.opencode.create_session(title=resource["name"])

    async def send_message(self, *, session_meta: dict, text: str, user: dict, trace_id: str) -> dict:
        system = build_skill_system_prompt(
            skill_name=session_meta["skill_name"],
            desc=session_meta.get("skill_desc", "")
        )
        return await self.opencode.send_message(
            session_id=session_meta["engine_session_id"],
            text=text,
            system=system
        )

    async def list_messages(self, *, session_meta: dict, trace_id: str) -> list[dict]:
        return await self.opencode.list_messages(session_meta["engine_session_id"])
```

这里的关键点是：Skill 卡片、权限、入口由 Portal 控；真正的 skill 执行仍交给 OpenCode 原生 skill 机制。OpenCode 官方文档明确写了 skill 是从 `SKILL.md` 发现，并由原生 `skill` 工具按需加载。([OpenCode][2])

## 3）WebSDKAdapter

```python
# app/adapters/websdk.py
import time
import jwt
from urllib.parse import urlencode, urlsplit, urlunsplit, parse_qsl
from app.models import Resource, UserCtx

LAUNCH_SECRET = "replace-me"

def append_query(url: str, extra: dict[str, str]) -> str:
    parts = list(urlsplit(url))
    query = dict(parse_qsl(parts[3]))
    query.update(extra)
    parts[3] = urlencode(query)
    return urlunsplit(parts)

class WebSDKAdapter:
    async def launch(self, *, resource: Resource, user: UserCtx, trace_id: str) -> dict:
        launch_token = jwt.encode(
            {
                "emp_no": user.emp_no,
                "resource_id": resource.id,
                "trace_id": trace_id,
                "iat": int(time.time()),
                "exp": int(time.time()) + 600
            },
            LAUNCH_SECRET,
            algorithm="HS256"
        )

        # V1 兼容期：既传 launch_token，也可选传 emp_no
        base_url = append_query(resource.config["base_url"], {
            "launch_token": launch_token,
            "emp_no": user.emp_no,
            "trace_id": trace_id
        })

        return {
            "kind": "websdk",
            "script_url": resource.config["script_url"],
            "app_key": resource.config["app_key"],
            "base_url": base_url
        }
```

## 4）OpenWorkAdapter

OpenWork server 当前 README 已经列出 `/workspace/:id/skills`、`PATCH /workspace/:id/config`、`POST /workspace/:id/engine/reload` 等 endpoint，而且官方架构文档明确要求文件系统变更尽量都走 server。([GitHub][4])

```python
# app/adapters/openwork.py
import httpx

class OpenWorkAdapter:
    def __init__(self, base_url: str, token: str):
        self.base_url = base_url.rstrip("/")
        self.headers = {"Authorization": f"Bearer {token}"}

    async def list_skills(self, workspace_id: str) -> list[dict]:
        async with httpx.AsyncClient(timeout=30, headers=self.headers) as client:
            resp = await client.get(f"{self.base_url}/workspace/{workspace_id}/skills")
            resp.raise_for_status()
            return resp.json()

    async def reload_engine(self, workspace_id: str) -> bool:
        async with httpx.AsyncClient(timeout=30, headers=self.headers) as client:
            resp = await client.post(f"{self.base_url}/workspace/{workspace_id}/engine/reload")
            resp.raise_for_status()
            return True
```

@app.get("/api/resources")
async def list_resources(user = Depends(get_current_user)):
    items = [r for r in catalog.list_all() if can_access(user, r)]
    return [x.model_dump() for x in items]

@app.get("/api/skills")
async def list_skills(user = Depends(get_current_user)):
    items = [r for r in catalog.list_all() if r.type == ResourceType.SKILL and can_access(user, r)]
    # 可选：和 OpenWork 实际安装状态合并
    installed = {x.get("name") for x in await openwork.list_skills("default")}
    result = []
    for item in items:
        skill_name = item.config["skill_name"]
        result.append({
            **item.model_dump(),
            "installed": skill_name in installed
        })
    return result

@app.post("/api/resources/{resource_id}/launch")
async def launch_resource(resource_id: str, user = Depends(get_current_user)):
    resource = catalog.get(resource_id)
    if not resource:
        raise HTTPException(404, "resource not found")
    if not can_access(user, resource):
        raise HTTPException(403, "forbidden")

    trace_id = str(uuid.uuid4())
    now = int(time.time())

    if resource.type in (ResourceType.DIRECT_CHAT, ResourceType.SKILL):
        engine_session = await opencode.create_session(title=resource.name)
        portal_session_id = str(uuid.uuid4())

        session = PortalSession(
            id=portal_session_id,
            emp_no=user.emp_no,
            resource_id=resource.id,
            engine_type="opencode",
            engine_session_id=engine_session["id"],
            title=resource.name,
            skill_name=resource.config.get("skill_name"),
            created_at=now,
            updated_at=now
        )
        await store.save_session(session)

        return {
            "kind": "native",
            "portal_session_id": portal_session_id,
            "resource": resource.model_dump()
        }

    launch_payload = await websdk.launch(resource=resource, user=user, trace_id=trace_id)
    launch = LaunchRecord(
        id=str(uuid.uuid4()),
        emp_no=user.emp_no,
        resource_id=resource.id,
        resource_type=resource.type.value,
        trace_id=trace_id,
        created_at=now,
        launch_kind="websdk",
        payload=launch_payload
    )
    await store.save_launch(launch)

    return {
        "kind": "websdk",
        "launch_id": launch.id,
        "resource": resource.model_dump()
    }

@app.get("/api/launches/{launch_id}/embed-config")
async def get_embed_config(launch_id: str, user = Depends(get_current_user)):
    launch = await store.get_launch(launch_id)
    if not launch or launch.emp_no != user.emp_no:
        raise HTTPException(404, "launch not found")
    return launch.payload

@app.get("/api/sessions")
async def list_sessions(user = Depends(get_current_user)):
    ids = await store.list_user_sessions(user.emp_no)
    sessions = []
    for sid in ids:
        meta = await store.get_session(sid)
        if meta:
            sessions.append(meta.model_dump())
    return sessions

@app.get("/api/sessions/{portal_session_id}/messages")
async def get_messages(portal_session_id: str, user = Depends(get_current_user)):
    meta = await store.get_session(portal_session_id)
    if not meta or meta.emp_no != user.emp_no:
        raise HTTPException(404, "session not found")
    return await opencode.list_messages(meta.engine_session_id)

@app.post("/api/sessions/{portal_session_id}/messages")
async def send_message(portal_session_id: str, body: dict, user = Depends(get_current_user)):
    meta = await store.get_session(portal_session_id)
    if not meta or meta.emp_no != user.emp_no:
        raise HTTPException(404, "session not found")

    text = body["text"]
    trace_id = str(uuid.uuid4())

    if meta.skill_name:
        result = await skill_chat.send_message(
            session_meta=meta.model_dump(),
            text=text,
            user=user.model_dump(),
            trace_id=trace_id
        )
    else:
        result = await opencode.send_message(
            session_id=meta.engine_session_id,
            text=text
        )

    return result

## 七、websdk 的前端实现

## 1）为什么我建议用“独立 html host 页”

你给的 websdk 代码是直接往 html body 里塞 script 并初始化。
这类 SDK 往往不是为 React/Vue 组件化容器设计的，所以 V1 最稳的做法不是在 SPA 里直接 mount，而是：

* Portal 主应用里放一个 iframe
* iframe 指向一个专门的 `sdk-host.html`
* `sdk-host.html` 再去拉取 launch config 并注入 SDK script

这样 SDK 污染不到主应用 DOM，也更容易隔离样式/全局变量。

## 2）sdk-host.html

```html
<!-- public/sdk-host.html -->
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8" />
  <title>SDK Host</title>
  <style>
    html, body { margin: 0; padding: 0; width: 100%; height: 100%; overflow: hidden; }
  </style>
</head>
<body>
  <script>
    (async function () {
      const qs = new URLSearchParams(location.search);
      const launchId = qs.get("launchId");

      const resp = await fetch(`/api/launches/${launchId}/embed-config`, {
        credentials: "include"
      });
      if (!resp.ok) {
        document.body.innerHTML = "<pre>load embed config failed</pre>";
        return;
      }

      const cfg = await resp.json();

      const script = document.createElement("script");
      script.src = cfg.script_url;
      script.onload = function () {
        // 按你给的 SDK 形式初始化
        new HiagentWebSDK.WebLiteClient({
          appKey: cfg.app_key,
          baseUrl: cfg.base_url
        });
      };
      document.body.appendChild(script);
    })();
  </script>
</body>
</html>
```

## 6）原生对话页

```tsx
// src/NativeChatPage.tsx
import React, { useEffect, useState } from "react";
import { getMessages, sendMessage } from "./api";

export function NativeChatPage({ portalSessionId }: { portalSessionId: string }) {
  const [messages, setMessages] = useState<any[]>([]);
  const [text, setText] = useState("");

  useEffect(() => {
    getMessages(portalSessionId).then(setMessages);
  }, [portalSessionId]);

  async function onSend() {
    if (!text.trim()) return;
    const userText = text;
    setText("");

    setMessages(prev => [...prev, { info: { role: "user" }, parts: [{ type: "text", text: userText }] }]);

    const resp = await sendMessage(portalSessionId, userText);
    // 这里可以按 OpenCode 返回结构做 normalize
    setMessages(prev => [...prev, resp]);
  }

  return (
    <div className="chat-page">
      <div className="messages">
        {messages.map((m, i) => (
          <pre key={i}>{JSON.stringify(m, null, 2)}</pre>
        ))}
      </div>
      <div className="input-bar">
        <textarea value={text} onChange={e => setText(e.target.value)} />
        <button onClick={onSend}>发送</button>
      </div>
    </div>
  );
}
```

## 八、Skill 商店的 V1 做法

V1 不做编辑，所以 Skill 商店建议：

* **Portal catalog**：管展示、分组、权限、描述、starter prompts
* **OpenWork server**：管“这个 skill 当前是否已安装在 workspace”
* **OpenCode**：真正运行 skill

实现上就是：

1. `/api/skills` 返回 Portal 可见 skill
2. 再用 OpenWork `GET /workspace/:id/skills` 标记 `installed`
3. 用户点击后直接 `POST /api/resources/{id}/launch`
4. 发消息时自动进入 skill mode

这版已经满足你要的“可用可控”。
不做编辑，也不要求 Portal 直接读写 `SKILL.md`。

## 九、日志与 trace

OpenWork 官方 orchestrator 文档已经说明可以输出统一日志流、JSON 格式、稳定 run id 关联，并且 OpenWork server 会记录 request 的 method/path/status/duration。你自己的 Portal 最好沿用同样的思路。([GitHub][5])

建议所有请求统一打这些字段：

```python
# app/logging.py
import time, uuid, json
from starlette.middleware.base import BaseHTTPMiddleware

class TraceMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        trace_id = request.headers.get("X-Trace-Id") or str(uuid.uuid4())
        start = time.time()
        response = await call_next(request)
        cost = int((time.time() - start) * 1000)

        log = {
            "trace_id": trace_id,
            "path": request.url.path,
            "method": request.method,
            "status": response.status_code,
            "cost_ms": cost,
        }
        print(json.dumps(log, ensure_ascii=False))
        response.headers["X-Trace-Id"] = trace_id
        return response
```

### 建议 trace 维度

* `trace_id`
* `emp_no`
* `resource_id`
* `portal_session_id`
* `engine_session_id`
* `launch_id`
* `adapter`
* `cost_ms`

## 十一、关键流程伪代码

## 1）登录流程

```text
User -> /login
  -> mock-sso or future real sso
  -> callback(code)
  -> resolve_emp_no_from_sso(code)
  -> sign portal_token
  -> set cookie
  -> redirect /
```

## 2）启动原生对话

```text
click resource
  -> POST /api/resources/{id}/launch
  -> ACL check
  -> OpenCode create session
  -> save PortalSession(redis)
  -> return kind=native + portal_session_id
  -> frontend open NativeChatPage
```

## 3）启动 Skill 对话

```text
click skill
  -> POST /api/resources/{id}/launch
  -> ACL check
  -> OpenCode create session
  -> save PortalSession(skill_name=xxx)
  -> frontend open NativeChatPage
  -> send message
       -> backend sees skill_name
       -> inject system prompt
       -> call OpenCode /session/:id/message
```

## 4）启动 websdk 知识库 / 旧应用

```text
click resource
  -> POST /api/resources/{id}/launch
  -> ACL check
  -> sign launch_token
  -> save LaunchRecord(redis)
  -> return launch_id
  -> frontend open iframe(/sdk-host.html?launchId=xxx)
  -> sdk-host loads config
  -> inject script + init SDK
```

## 十二、V1 第一轮开发顺序

我建议按这个顺序做，最稳：

1. `mock auth + portal_token`
2. `resource catalog + ACL`
3. `OpenCodeAdapter + 原生对话`
4. `PortalSession + Redis`
5. `SkillChatAdapter + Skill 商店`
6. `WebSDKAdapter + sdk-host.html`
7. `LaunchRecord + 最近打开`
8. `OpenWorkAdapter + skill installed 状态`
9. `trace middleware + JSON log`
